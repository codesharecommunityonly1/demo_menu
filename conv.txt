## Goal

Build a 3D Restaurant Menu web application for Hugging Face Spaces using Gradio. The app should display Indian food items with categories, search (with smart suggestions for typos), price filtering, and interactive 3D model viewing using Three.js.

## Instructions

- Create a Gradio app for Hugging Face deployment
- Include menu items with categories, prices, descriptions, and images
- Add search functionality with suggestions for wrong spellings/typos
- Add price filter dropdown (Low to High / High to Low)
- Add category filter dropdown
- Add 3D model viewing with Three.js when clicking "View 3D"
- Make the UI modern, stylish, and attractive with gradients and animations
- Create folders for images and 3D models with proper README files listing all item IDs
- Load images from local images folder (not URLs) once images are added

## Discoveries

- Hugging Face automatically adds `gradio[oauth]==5.0.0` which conflicts with custom version requirements in requirements.txt
- Several Gradio versions (4.x, 5.x) had `pyaudioop` module issues with Python 3.13 - not available in standard library
- Solution: Remove requirements.txt entirely and let Hugging Face use its default gradio version
- Three.js loaded via CDN for 3D model rendering in browser

To run this Gradio app on a new computer:

```bash
# 1. Navigate to the project folder
cd path/to/demo_menu

# 2. Create virtual environment (optional but recommended)
python -m venv venv

# 3. Activate virtual environment
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 4. Install gradio
pip install gradio

# 5. Run the app
python app.py
```

The app will start at `http://localhost:7860` in your browser.

If you want to open it automatically:
```bash
python app.py --share
```
